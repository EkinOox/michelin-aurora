La problématique client
03
Brief — Michelin LB 2 Wheels
"Donner de la visibilité à Michelin LB 2 Wheels auprès d'une clientèle premium. Créer de l'appétence pour la marque en offrant une gamme de services innovants."
— Abdellatif Ghachi, Global Account Manager E-retail 2W
Problème n°1 — Visibilité
Pas de point de contact digital
Michelin LB 2 Wheels vend des pneus premium, mais n'a aucune app ni service numérique. La marque disparaît après l'achat.
Problème n°2 — Appétence
Aucun service à valeur ajoutée
Le cycliste premium attend un partenaire de performance, pas seulement un consommable. Sans service innovant, Michelin reste une marque parmi d'autres.
Aurora place Michelin au cœur de chaque sortie cycliste — pression intelligente, télémétrie live, itinéraires, fidélisation. Le pneu devient un service actif et visible au quotidien.

Choix techniques clés
07
SSE plutôt que WebSocket
Télémétrie unidirectionnelle serveur → client. Natif HTTP, aucune config proxy. Idéal pour simuler un capteur pneu connecté.
Nuxt 4 SSR hybride
Premier paint côté serveur (SEO + perf), puis client-side. useApiFetch gère le JWT en SSR et CSR automatiquement.
Algo pression multi-facteurs
Bases par discipline + ajustements dynamiques : température, précipitations WMO, vent, niveau rider. Facteurs exposés dans l'API.
JWT stateless
Authentification sans session serveur. Scalable, compatible PWA offline et partage du token entre Nuxt SSR et requêtes client.
PWA mobile-first
Installable iOS/Android. Bottom sheets via Teleport + position:fixed. Safe-area iPhone. dvh Safari. Géolocalisation cachée localStorage.
Cache API stratifié
GPS : 1h · Géométrie OSM : 7j · API routes : sessionStorage. Zéro re-demande de permission GPS pendant la sortie.

Ce que nous pourrions améliorer
12
Tracking GPS live
Tracé de route en temps réel sur la carte pendant la sortie — courbe de pression superposée au profil altimétrique.
Recommandations ML
Modèle de Machine Learning entraîné sur l'historique des sorties pour prédire la pression optimale avant même le départ.
Capteur physique BLE
Intégration d'un vrai module Bluetooth Low Energy sur le pneu pour remplacer le simulateur Python par des données réelles.
Analyse post-sortie
Graphiques de pression, vitesse et altitude sur toute la sortie. Détection automatique des segments difficiles.
Intégration Strava / Komoot
Import automatique des itinéraires et synchronisation des sorties avec les plateformes cyclistes déjà utilisées par la communauté.
Marketplace intégrée
Achat direct de pneus Michelin dans l'app avec livraison chez le revendeur le plus proche — boucle commerciale complète.
Ces évolutions sont déjà identifiées dans notre backlog. L'architecture Aurora — API REST, SSE, composables Nuxt — est conçue pour les accueillir sans refonte majeure.
